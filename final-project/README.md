## Итоговая работа

Проект является учебным и ставит перед собой цель продемонстрировать работу 
с различными технологиями.

### Предметная область
В качестве темы выбран сервис для управления задачами. Имеется регистрация и авторизация сотрудника. 
После входа в систему сотрудник может посмотреть свои задачи, завести новые, прикрепить их к себе или наоборот снять с себя.
При заведении новой задачи можно выбрать, сразу ли повесить задачу на себя или же просто создать её.

### Структура
Проект реализован с помощью технологии `Spring Boot` на микросервисной архитектуре и состоит из нескольких модулей:

1. Модуль `commons-spring-boot-starter` - вспомогательный модуль-стартер, в который вынесена общая логика 
для других модулей (например, обработка ошибок или `AOP`);
   
2. Модуль `database-service` - модуль, в который помещена система управления версиями базы данных `Liquibase`;

3. Модуль `employee-service` - сервис для работы с сотрудниками и близко связанными с ними сущностями. Полноценное
`Web-MVC` приложение;
   
4. Модуль `eureka-service` - `service discovery` приложение для регистрации других сервисов и обеспечения общения между ними;
Благодаря этому сервису, например, клиентским приложениям (модулям) не нужно знать точный адрес другого модуля, 
   к которому требуется обратиться;

5. Модуль `front-service` - приложение, выполняющее роль `frontend` части всего проекта. Реализовано с помощью 
`Spring Boot` и `Thymeleaf`;
   
6. Модуль `kafka-service` - вспомогательный модуль, в который помещена конфигурация для обмена сообщениями посредством `Apache Kafka`;

7. Модуль `task-service` - сервис для работы с задачами и близко связанными с ними сущностями. Полноценное 
`Web-MVC` приложение.
